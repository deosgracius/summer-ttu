# Summer (TTU ECE campus assistant) — project rules

Authoritative rules for everyone working in this repo, human or AI. These are
decisions already made for this project; follow them unless the owner changes them.
Security specifics live in [SECURITY.md](SECURITY.md).

## Hard rules (non-negotiable)

1. **No surveillance, ever.** Summer never tracks, records, requests, or infers
   anyone's physical location, and never uses GPS, geolocation, or maps to locate a
   person — including tutors and staff. No geofencing, no location-restricted
   check-ins, no monitoring of whereabouts or activity. A tutor time-clock, if built,
   is voluntary in-person kiosk check-in only (name + timestamp the person chose to
   enter), verified by PIN or QR, never by location. This is baked into the agent
   prompts; keep it there.

2. **Privacy of sensitive data.** Advising/registration records are managed by the
   school's own backend, not by Summer. Summer is a campus information assistant.
   Never store, track, or infer personal information beyond what a feature strictly
   needs and the user has consented to.

3. **Provenance, never fabrication.** Authoritative campus facts (room, office, time,
   instructor, prerequisite, policy, deadline, number) must be RETRIEVED from a tool
   or the DB and reflect that source — never generated by the model. If the data has
   no answer, say so and point to who to contact. Do not invent.

4. **Security-review every change.** Threat-model each change for authz, injection,
   secrets exposure, XSS, SSRF, and broken access control before shipping. See
   SECURITY.md.

5. **Least privilege.** The public kiosk is anonymous, read-only, and hard-restricted
   to campus lookup tools. Privileged tools are central-admin gated and granted
   per-user. Don't widen these without a reason.

6. **Graceful degradation.** When the LLM is unreachable (no credits, outage), Summer
   falls back to a deterministic DB lookup — it must never crash at the user.

7. **Plain, professional output.** Summer's replies are plain text: no markdown
   symbols (`**`, `#`, backticks, `-` bullets), no emoji, no casual openers. The chat
   surface does not render markdown.

8. **Cost-sensitive by default.** Flag real costs up front and lead with free options.
   Route deterministic lookups before paying the LLM. Put spend caps on paid APIs.

9. **Secrets.** Real secrets live only in `.env` (gitignored, untracked) and in Fly
   secrets for production. Never commit, print, or echo a secret. `.env` must never be
   tracked.

10. **Original stays untouched.** This repo is the TTU clone. The original `summer_app`
    project must not be modified.

## Working agreements

- **Concurrent sessions.** Multiple sessions may edit this repo at the same time. Make
  minimal, localized changes; verify the working tree before committing; and do not
  revert another session's intentional changes. Review the full diff before staging.
- **Tests/CI are a gate.** Keep the structural eval gate and the deterministic test
  suite green. The LLM-vs-search-box benchmark's deterministic checks run in CI
  (`tests/test_benchmark.py`); the live benchmark is opt-in
  (`RUN_LIVE_EVALS=1` + an API key). Run `python -m pytest -q` before committing.
- **Data imports** (e.g. `import_ttu_ece.py`) pull only public pages and are
  re-runnable. Writes go to the live Neon DB, so production sees them immediately.
